블랙잭 게임 제작 요청 프롬프트 (Railway 배포 + 관리자 시스템)

당신은 숙련된 풀스택 웹 개발자이자 UI/UX 디자이너입니다. 저는 당신에게 Railway(railwayup.com)에 바로 배포 가능한 완성도 높은 블랙잭(Blackjack) 웹 애플리케이션을 제작해 달라고 요청드립니다. 단순한 프론트엔드 게임이 아니라, 백엔드 서버와 데이터베이스를 갖춘 실제 서비스 수준의 풀스택 애플리케이션입니다. 아래의 요구사항을 철저히 반영하여 제작해 주세요.

🏗️ 기술 스택 및 프로젝트 구조
백엔드:

Runtime: Node.js (v18 이상)
Framework: Express.js
Database: SQLite3 (better-sqlite3 패키지, 파일 기반으로 Railway 볼륨에 저장)
인증: express-session + bcryptjs (관리자 로그인용)
기타: cors, dotenv

프론트엔드:

순수 HTML5, CSS3, Vanilla JavaScript
외부 UI 라이브러리 없이 직접 구현
백엔드 API와 fetch()로 통신

프로젝트 파일 구조:

👤 플레이어 시스템 (회원가입 없는 이름 기반 저장)
핵심 컨셉:
회원가입, 이메일, 비밀번호 없이 이름(닉네임)만 입력하면 자동으로 플레이어 계정이 생성 또는 불러와지는 구조입니다.
동작 방식:

접속 시 이름 입력 화면이 표시됩니다.
플레이어가 이름을 입력하고 "게임 시작" 버튼을 누릅니다.
해당 이름이 DB에 없으면 → 새 플레이어로 자동 생성 (초기 칩: 0, 잔액: $0)
해당 이름이 DB에 있으면 → 기존 데이터를 불러와서 이어서 플레이
이름은 브라우저 localStorage에도 저장하여 다음 방문 시 자동 입력
같은 이름을 입력하면 항상 같은 계정으로 접속 (이름 = 고유 식별자)
플레이어 이름은 2~20자, 한글/영문/숫자만 허용

저장 데이터 (players 테이블):
sqlCREATE TABLE players (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT UNIQUE NOT NULL,           -- 플레이어 이름 (고유 식별자)
  chips INTEGER DEFAULT 0,             -- 현재 보유 칩 수
  balance INTEGER DEFAULT 0,           -- 현재 잔액 (달러 cents 단위로 저장)
  total_games INTEGER DEFAULT 0,       -- 총 게임 수
  total_wins INTEGER DEFAULT 0,        -- 총 승리 수
  total_losses INTEGER DEFAULT 0,      -- 총 패배 수
  total_pushes INTEGER DEFAULT 0,      -- 총 무승부 수
  biggest_win INTEGER DEFAULT 0,       -- 최대 단판 승리 금액
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  last_played DATETIME DEFAULT CURRENT_TIMESTAMP
);

🔑 관리자 시스템
관리자 접속:

URL: /admin 페이지로 접속
아이디/비밀번호로 로그인 (환경변수로 설정: ADMIN_ID, ADMIN_PASSWORD)
세션 기반 인증, 로그인 유지 시간 24시간
기본값: ID admin / PW admin1234 (반드시 환경변수로 변경 권장)

관리자 기능:

플레이어 목록 조회: 전체 플레이어 목록, 잔액, 칩, 게임 통계 한눈에 보기
칩 지급: 특정 플레이어에게 칩을 지급 (예: +100칩, +500칩 등 직접 입력)
칩 차감: 특정 플레이어의 칩을 차감
잔액 직접 설정: 플레이어 잔액을 특정 금액으로 설정
플레이어 초기화: 특정 플레이어의 칩/잔액을 0으로 초기화
플레이어 삭제: 특정 플레이어 계정 완전 삭제
전체 통계: 총 플레이어 수, 오늘 접속자 수, 전체 게임 수, 전체 칩 발행량
칩 지급 내역 로그: 언제 누구에게 얼마의 칩을 지급했는지 기록

칩 지급 로그 테이블:
sqlCREATE TABLE chip_logs (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  player_name TEXT NOT NULL,
  action TEXT NOT NULL,        -- 'add' / 'subtract' / 'set' / 'reset'
  amount INTEGER NOT NULL,     -- 지급/차감 량
  before_chips INTEGER,        -- 변경 전 칩
  after_chips INTEGER,         -- 변경 후 칩
  admin_note TEXT,             -- 관리자 메모 (선택)
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

🎮 게임 로직 (서버사이드 검증 포함)
중요: 게임의 핵심 로직(카드 덱, 딜링, 승패 판정)은 서버에서 처리하고 결과만 클라이언트로 전달합니다. 클라이언트에서 잔액/칩을 직접 조작할 수 없도록 합니다.
게임 세션 흐름 (API):
POST /api/game/start     → 게임 시작, 덱 생성, 초기 카드 배분
POST /api/game/hit       → 플레이어 히트, 카드 추가
POST /api/game/stand     → 플레이어 스탠드, 딜러 플레이
POST /api/game/double    → 더블다운
POST /api/game/split     → 스플릿
POST /api/game/bet       → 베팅 설정
POST /api/game/result    → 최종 결과 처리 및 DB 저장
게임 규칙:

표준 52장 덱 사용, 덱 25% 이하 남으면 자동 셔플
숫자 카드 액면가, J/Q/K=10, Ace=1 또는 11 자동 계산
딜러 두 번째 카드 숨김 처리
플레이어: Hit, Stand, Double Down, Split 지원
버스트 시 즉시 패배
딜러는 16 이하 Hit, 17 이상 Stand
블랙잭(첫 2장=21) 시 1.5배 페이아웃
플레이어는 보유 칩 범위 내에서만 베팅 가능

베팅 단위: 1칩 = $1 로 처리 (칩이 곧 게임 화폐)

🎨 UI/UX 디자인 요구사항
플레이어 화면 - 이름 입력 페이지 (/):

어두운 카지노 배경
중앙에 게임 로고 + 이름 입력창
"처음이신가요? 이름을 입력하면 자동으로 계정이 만들어집니다" 안내 문구
기존 방문자라면 localStorage에서 이름 자동 채워주기
입력 후 "게임 입장" 버튼

플레이어 화면 - 게임 페이지 (/game):

상단 헤더: 플레이어 이름, 현재 칩 수, 현재 잔액 표시
중앙: 딜러 영역 (카드 + 점수)
중앙: 플레이어 영역 (카드 + 점수)
하단: 베팅 패널 (칩 단위 버튼: 1, 5, 25, 100), 현재 베팅액 표시
하단: 액션 버튼 (Hit / Stand / Double / Split)
우측 사이드: 게임 히스토리 (최근 10판), 내 통계

관리자 화면 (/admin):

깔끔한 대시보드 스타일
플레이어 테이블 (이름, 칩, 잔액, 총 게임, 승률, 마지막 접속)
각 플레이어 행에 [칩 지급] [칩 차감] [초기화] [삭제] 버튼
칩 지급 시 모달창에서 금액 + 메모 입력
실시간 검색으로 플레이어 필터링
전체 통계 카드 (총 플레이어, 오늘 접속, 총 게임 수)

공통 디자인:

어두운 초록 펠트(felt) 카지노 테마
금색 포인트 컬러
카드 딜링 애니메이션 (CSS 3D transform)
모바일 반응형 완전 지원
카드 앞/뒷면 플립 애니메이션


⚙️ Railway 배포 설정
package.json scripts:
json{
  "scripts": {
    "start": "node server.js",
    "dev": "nodemon server.js"
  }
}
railway.toml:
toml[build]
builder = "nixpacks"

[deploy]
startCommand = "npm start"
restartPolicyType = "on_failure"
환경변수 (.env.example):
PORT=3000
SESSION_SECRET=your-secret-key-here
ADMIN_ID=admin
ADMIN_PASSWORD=admin1234
DB_PATH=./database/blackjack.db
Railway 볼륨 마운트:

./database 폴더를 Railway 퍼시스턴트 볼륨에 마운트하여 SQLite DB가 재배포 후에도 유지되도록 설정
DB_PATH 환경변수로 DB 경로 지정


📋 API 명세
플레이어 API (/api/player):
POST   /api/player/login         { name } → 플레이어 생성 또는 조회
GET    /api/player/:name         플레이어 정보 조회
GET    /api/player/:name/history 게임 히스토리 조회
게임 API (/api/game):
POST   /api/game/start           { playerName, betAmount } → 게임 시작
POST   /api/game/action          { playerName, action: 'hit'|'stand'|'double'|'split' }
GET    /api/game/state/:playerName 현재 게임 상태 조회
관리자 API (/api/admin):
POST   /api/admin/login          관리자 로그인
POST   /api/admin/logout         관리자 로그아웃
GET    /api/admin/players        전체 플레이어 목록
POST   /api/admin/chips          { playerName, action, amount, note } → 칩 조작
GET    /api/admin/logs           칩 지급 로그 조회
GET    /api/admin/stats          전체 통계
DELETE /api/admin/player/:name   플레이어 삭제
